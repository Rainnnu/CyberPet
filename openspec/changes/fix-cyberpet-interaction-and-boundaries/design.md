## Context

当前桌面宠物应用使用Electron + React + Three.js构建。窗口设置为透明、无边框、始终置顶，并使用`WebkitAppRegion: 'drag'`实现拖拽功能。然而存在两个问题：

1. 点击和右键事件处理存在问题，导致对话框和设置面板无法正常显示
2. 窗口拖拽没有边界限制，可以被拖到屏幕外

## Goals / Non-Goals

**Goals:**
- 恢复点击宠物弹出对话框的功能
- 恢复右键菜单显示设置面板的功能
- 添加窗口拖拽边界限制，确保窗口始终在屏幕可视区域内
- 保持现有的窗口样式和透明效果

**Non-Goals:**
- 不改变现有的3D渲染逻辑
- 不修改宠物模型和动画
- 不改变现有的AI对话功能

## Decisions

### 1. 交互事件处理方案

**问题分析**：当前App.tsx中，3D场景和宠物覆盖层都设置了`pointerEvents: 'none'`，这可能导致事件无法正确传递。

**解决方案**：
- 保持3D场景的`pointerEvents: 'none'`（因为Canvas有自己的事件系统）
- 移除宠物覆盖层的`pointerEvents: 'none'`，使其可以接收点击事件
- 确保`WebkitAppRegion: 'drag'`不影响点击事件的传递

**备选方案**：
- 使用Three.js的原生点击事件系统（需要重构较多代码）
- 使用Electron的IPC通信处理点击事件（过于复杂）

**选择原因**：当前方案最简单，只需修改CSS属性即可解决问题。

### 2. 窗口边界限制方案

**问题分析**：Electron的`BrowserWindow`没有内置的边界限制功能，窗口可以自由拖拽到任何位置。

**解决方案**：
- 在主进程中监听窗口的`moved`事件
- 在窗口移动时检查位置是否超出屏幕边界
- 如果超出边界，将窗口位置调整回屏幕内
- 使用`screen.getPrimaryDisplay().workAreaSize`获取可用屏幕区域

**备选方案**：
- 使用`screen.getCursorScreenPoint()`限制鼠标位置（会影响其他应用）
- 使用第三方库`electron-window-state`（增加依赖）

**选择原因**：直接监听窗口移动事件最简单，且不影响其他应用。

### 3. 边界检测时机

**问题分析**：需要在窗口移动过程中实时检测边界，而不是只在移动结束后检测。

**解决方案**：
- 使用`requestAnimationFrame`或`setInterval`定期检查窗口位置
- 或者使用Electron的`will-move`事件（如果可用）

**选择原因**：使用`moved`事件结合实时检查，确保用户体验流畅。

## Risks / Trade-offs

### 风险1：事件传递问题
- **风险**：修改`pointerEvents`可能影响现有的拖拽功能
- **缓解**：测试拖拽和点击功能是否正常工作

### 风险2：性能影响
- **风险**：频繁的边界检查可能影响性能
- **缓解**：使用节流（throttle）控制检查频率，只在窗口移动时检查

### 风险3：多显示器支持
- **风险**：当前方案只考虑主显示器
- **缓解**：后续可以扩展支持多显示器

### 风险4：窗口状态保存
- **风险**：边界调整后的窗口位置可能与保存的位置不一致
- **缓解**：在边界调整后更新保存的配置
